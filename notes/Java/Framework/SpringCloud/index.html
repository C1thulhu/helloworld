#分布式系统

  在介绍SpringCloud之前我们需要先了解什么是分布式系统，简单来说就是一群计算机集合共同对外提供服务，它们彼此之间仅通过消息传递进行通信和协调。计算机越多，CPU、内存、储存空间等资源也越多，能够处理的并发访问量也越大。

主要特征

  • 分布性

    分布式系统的多台计算机空间位置可以随意分布，机器分布情况也可随时变动

  • 对等性

    分布式系统计算机没有从属关系，所有计算机节点都是对等的

  • 自治性

    分布式系统中各个节点包含各自的处理器与内存，都具有独立的处理数据功能。能自治地进行工作又能利用共享通信线路来传递信息，协调任务处理。

  • 并发性

    在一个计算机网络中，程序运行过程的并发性操作是非常常见的行为。同一个分布式系统中的多个节点，可能会并发地操作一些共享的资源，如何准确并高效地协调分布式并发操作也成为了分布式系统架构与设计中最大的挑战之一。

#SpringCloud介绍

  SpringCloud是一系列框架的有序集合体，它是建立在Spring Boot的开发便利性上巧妙地简化了分布式系统基础设施的开发。SpringCloud只是将已经成熟的服务框架组合，通过SpringBoot风格再封装，屏蔽了复杂的配置和实现原理，最终给开发者提供了一套简单易懂、易部署、易维护的分布式系统开发工具包。

  Spring Cloud 本身并不是一个拿来即可用的框架，它是一套微服务规范，目前共有两代实现。Spring Cloud 中包含了近20个子项目，提供了服务注册、服务网关、智能路由、负载均衡、断路器、监控跟踪、分布式消息队列、配置管理等具体问题的解决方案。

  • Spring Cloud Netflix 是 Spring Cloud 的第一代实现，主要由 Eureka、Ribbon、Feign、Hystrix 等组件组成。
  • Spring Cloud Alibaba 是 Spring Cloud 的第二代实现，主要由 Nacos、Sentinel、Seata 等组件组成。

  2018年12月12日，Netflix公司宣布Spring Cloud Netflix系列大部分组件停更进入维护模式，不再添加新特性。Spring Cloud Alibaba吸收Spring Cloud Netflix核心架构思想，并进行了高性能改进。自 Spring Cloud Netflix 进入停更维护后，Spring Cloud Alibaba 逐渐代替它成为主流的微服务框架。
本篇只针对Spring Cloud Alibaba做详细介绍。

Spring Cloud两代实现组件对比：

组件功能 Spring Cloud Netflix 优缺 Spring Cloud Alibaba 优缺
服务注册 Ereka 2.0 孵化失败 Nacos Discovery 性能更好，感知力更强
负载均衡 Ribbon 停更进维 Spring Cloud Loadbalancer Spring Cloud 原生组件，用于代替 Ribbon
断路器 Hystrix 停更进维 Sentinel 可视化配置，上手简单
服务网关 Zuul 停更进维 Spring Cloud Gateway 性能为 Zuul 的 1.6 倍
配置管理 Spring Cloud Config 搭建过程复杂，约定过多，无可视化界面，上手难点大 Nacos Config 搭建过程简单，有可视化界面，配置管理更简单，容易上手

#Spring Cloud组件

Nacos

Nacos 是一个更易于帮助构建云原生应用的动态服务发现、配置和服务管理平台即：服务注册和配置管理的组合体。

Nacos特性

  • 服务发现

    Nacos 支持基于 DNS 和 RPC 的服务发现。当服务提供者使用原生 SDK、OpenAPI 或一个独立的 Agent TODO 向 Nacos 注册服务后，服务消费者可以在 Nacos 上通过 DNS TODO 或 HTTP&API 查找、发现服务。

  • 服务健康监测

    Nacos 提供对服务的实时健康检查，能够阻止请求发送到不健康主机或服务实例上。Nacos 还提供了一个健康检查仪表盘，能够帮助我们根据健康状态管理服务的可用性及流量。
