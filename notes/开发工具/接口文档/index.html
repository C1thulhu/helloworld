#OpenAPI

  OpenAPI 规范（OAS）在新窗口打开定义了一个标准的、语言无关的 RESTful API 接口规范，它可以同时允许开发人员和操作系统查看并理解某个服务的功能，而无需访问源代码，文档或网络流量检查（既方便人类学习和阅读，也方便机器阅读）。正确定义 OAS 后，开发者可以使用最少的实现逻辑来理解远程服务并与之交互。

此外，文档生成工具可以使用 OpenAPI 规范来生成 API 文档，代码生成工具可以生成各种编程语言下的服务端和客户端代码，测试代码和其他用例。

#接口文档

  • 什么是接口文档
      在项目开发汇总，web项目的前后端是分离开发的。应用程序的开发，需要由前后端工程师共同定义接口，编写接口文档，之后大家都根据这个接口文档进行开发，到项目结束前都要一直维护。

  • 功能与目的

      项目开发过程中前后端工程师有一个统一的文件进行沟通交流开发，项目维护或者项目人员更迭的时候，方便后期人员查看、维护。

  • 接口文档规范

    1. 请求方式：GET / PUT / POST / DELETE；
    2. url：接口的url路径；
    3. 请求参数：请求参数的字段名、字段类型、说明、备注、是否必填；
    4. 响应参数：响应编码、响应数据和响应数据说明 或者响应错误信息。

#接口文档管理工具

  在项目开发测试中，接口文档是贯穿始终的。前后端开发需要在开发前期进行接口定义并形成文档，QA在功能测试和接口测试的环节也需要依赖于这些接口文档进行测试。接口文档往往以最简单的静态文档的形态存在。然而在紧张的**敏捷开发模式下，随着版本迭代，很多接口发生了变化或者被废弃，而开发几乎不会在后期去更新这种静态文档。**QA人员阅读“过期”的接口文档是一件痛苦的事情，与开发的沟通成本不降反升。而这些不便于及时维护的静态文档，随着时间的推移最终无人问津。因此，我们需要找到一种长期可维护且轻量便捷的接口文档工具。

Postman

  Postman是一个接口测试工具，在做接口测试的时候，Postman相当于一个客户端，它可以模拟用户发起的各类HTTP请求，将请求数据发送至服务端，获取对应的响应结果，从而校验接口和数据。

  我们可以创建团队类型项目工作区再添加协作人便可以共同管理接口文档，然后在Postman中根据 模块/功能 创建对应的folder，校验接口后就已经基本可以算上一个接口的文档了。
   总而言之Postmaan作为一个接口测试工具，适合小体量团队使用并维护。

DOClever

  DOClever是一个可视化接口管理工具，可以分析接口结构，校验接口正确性， 围绕接口定义文档，通过一系列自动化工具提升我们的协作效率。DOClever功能如下：

  1. 可以对接口信息进行编辑管理，支持 https 和 https 协议，并且支持 query，body，json，raw，rest，formdata 的参数可视化编辑。并且 状态码，代码注入，markdown 文档等附加功能应有尽有。
  2. 接口调试运行，可以对参数进行加密，从md5 到 aes 一应俱全，返回参数与模型实时分析对比，给出不一致的地方，找出接口可能出现的问题，可以对接口运行的数据一键生成文档信息。
  3. mock1 的无缝整合，DOClever 自己就是一个 mock 服务器，当你把接口的开发状态设置成已完成，本地 mock 便会自动请求真实接口数据，否则返回事先定义好的 mock 数据。
  4. 支持 postman，rap，swagger 的导入，方便你做无缝迁移，同时也支持 html 文件的导出，方便离线浏览
  5. 项目版本和接口快照功能并行，你可以为一个项目定义 1.0，1.1，1.2 版本，并且可以自由的在不同版本间切换回滚
  6. 自动化测试
  7. 团队协作功能

①：mock测试就是在测试过程中，对于某些不容易构造或者不容易获取的对象，用一个虚拟的对象来创建以便测试的测试方法。

Swagger

  Swagger 是一个用于生成、描述和调用 RESTful 接口的 Web 服务。通俗的来讲，Swagger 就是将项目中所有（想要暴露的）接口展现在页面上，并且可以进行接口调用和测试的服务。

从上述 Swagger 定义我们不难看出 Swagger 有以下 3 个重要的作用：

  • 将项目中所有的接口展现在页面上，这样后端程序员就不需要专门为前端使用者编写专门的接口文档；
  • 当接口更新之后，只需要修改代码中的 Swagger 描述就可以实时生成新的接口文档了，从而规避了接口文档老旧不能使用的问题；
  • 通过 Swagger 页面，我们可以直接进行接口调用，降低了项目开发阶段的调试成本。

Swagger3完全遵循了 OpenAPI 规范。Swagger 官网地址：https://swagger.io/

Smart-Doc

为何会产生Smart-Doc这类工具？目前主流接口文档工具存在问题：

  1. 侵入性强，需要编写大量注解。如swagger；
  2. 强依赖性，如果项目不想使用该工具，业务代码无法编译通过；
  3. 代码解析能力弱，使用文档不齐全；
  4. 众多基于注释分析工具无法解析jar包中的注释，需要人工配置源码路径，无法满足DevOps构建场景；
  5. 部分工具无法支持多模板复杂代码分析。
